# COITD 当前功能概览

## 1. 目录与边界

- `CloudflareSpeedTest/` 保持上游只读状态。
- 当前自定义能力集中在 `scripts/` 及其配套文档中。
- 目录以当前可维护主流程为准，不再保留未完成或已废弃脚本。

## 2. 已实现功能

### 2.1 配置文件驱动启动 CFST

- 启动器文件：`scripts/cfst/cfst_config_runner.py`
- 支持通过配置文件组装并执行 CFST 命令。
- 支持从结果 CSV 自动提取优选 IP。

### 2.2 配置格式与参数能力

- 支持 `JSON` 与 `JSONC`（允许 `//`、`/* */` 注释）。
- 兼容带 BOM 的配置文件读取（`utf-8-sig`）。
- 支持 `cfst -h` 全量参数映射（`options` 节点）。
- 支持参数名严格校验（`strict_known_options`）。
- 支持参数别名：`httping_code` -> `httping-code`。

### 2.3 优选 IP 输出能力

- 支持通过 `best_ip_count` 控制输出数量（Top N）。
- 标准输出：
  - `BEST_IP=...`（第一名）
  - `BEST_IP_LIST=...`（当 `best_ip_count > 1`）
- 文件输出：
  - `best_ip_file` 写入前 N 个 IP（每行一个）。

### 2.4 日本地区机房优选模板

- 模板文件：`scripts/cfst/cfst_config.jp.jsonc`
- 已预置 `HTTPing + cfcolo 日本地区码` 过滤。
- 可通过 `options.f` 指向“特定 IP 文件”。

### 2.5 面向 Web 控制的适配层

- 新增任务控制器：`scripts/cfst/cfst_job_controller.py`
- 支持后台启动、状态查询、日志读取、任务停止、历史任务列表。
- `cfst_config_runner.py` 新增 `--summary-json` 与 `--print-summary-json`，支持结构化输出结果，便于前后端对接。
- 支持生成 cron 模板命令，便于后续接入 Linux 定时任务。

### 2.6 CFST Web 控制台

- 页面脚本：`scripts/webui/cfst_web_console.py` + `scripts/webui/index.html`
- 页面能力：配置选择、任务启动/停止、状态查询、日志刷新、历史任务展示、定时任务配置与清除。
- 使用说明：`Docs/cfst_web_console.md`

### 2.7 定时任务网页化配置

- 支持在 Web 页面直接选择并写入 Linux `crontab`：
  - 每天运行一次
  - 每小时运行一次
  - 每半小时运行一次
  - 每 N 小时运行一次
- 支持在 Web 页面查看：
  - 定时任务“设置日志”（记录设置是否成功）
  - 定时任务“执行日志”（记录定时触发执行情况）

### 2.8 DDNS 自动推送

- 脚本：`scripts/cfst_ddns/cfst_ddns.sh`
- 支持把 CFST 结果推送到阿里云 DNS（AliDNS）。
- 支持 `LINE` 多线路配置与 `PUSH_IP_COUNT` 多 IP 推送。
- 支持写入结构化 `summary`，供独立监控页展示。

### 2.9 DDNS 专用 Web 监控页

- 页面脚本：`scripts/cfst_ddns_web/ddns_web_server.py` + `scripts/cfst_ddns_web/index.html`
- 页面能力：查看 timer/service 状态、最近执行结果、运行历史与日志。

## 3. 关键配置项

- `best_ip_count`：输出并写入前 N 个优选 IP。
- `best_ip_file`：保存优选 IP 结果文件。
- `options.f`：指定输入 IP 文件（特定 IP 文件）。
- `options.httping` + `options.cfcolo`：地区机房过滤关键组合。
