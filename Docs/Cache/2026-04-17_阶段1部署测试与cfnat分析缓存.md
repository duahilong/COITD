## 5. cfnat.sh 对比分析结论（缓存）

分析对象：`https://raw.cmliussss.com/cfnat.sh`

### 5.1 它的定位

- 更偏向 `cfnat/colo` 的一键安装、配置、守护与启动脚本
- 支持多架构下载和交互菜单
- 通过 `-task` 参数提升并发能力

### 5.2 与本项目第一阶段的差异

- 不提供本项目冻结契约：
  - 无 `run-once/status/history/validate-config/self-check/version` 命令模型
  - 无 JSON Envelope 契约
  - 无 `state.json/history.jsonl/run.lock` 一致性模型
- 更像“服务管理脚本”，不是“采集执行器 + 数据契约层”

### 5.3 可借鉴点

1. 参数化较完整（地区、延迟、并发、IP 数量等）
2. 部署体验与架构适配做得较多

### 5.4 风险点

1. 供应链风险：直接下载并执行远程二进制  
2. 安全风险：多处 `curl -k`（跳过证书校验）  
3. 运维风险：`pkill -f`、`crontab` 覆盖式管理，缺少强一致状态  
4. 网络暴露：默认监听 `0.0.0.0:$port`  
5. 脚本逻辑存在细节缺陷（随机开关判断变量混用）
