# DDNS 迁移阶段总结（2026-04-19）

## 1. 阶段目标

将 `cfst_ddns` 从原 Cloudflare 推送方式改为阿里云 DNS（AliDNS）推送，并在测试服务器完成端到端验证。

## 2. 本阶段已完成内容

1. 目录与边界
- 明确 `CloudflareSpeedTest` 目录为只读目录，仅用于读取，不做修改写入。
- 为定制开发建立独立工作目录：`scripts/cfst_ddns/`。

2. 脚本迁移
- 新版脚本：`scripts/cfst_ddns/cfst_ddns.sh`。
- 配置文件：`scripts/cfst_ddns/cfst_ddns.conf`、`scripts/cfst_ddns/cfst_ddns.conf.example`。
- 主要能力：
  - 执行 CFST 并读取测速结果首个 IP。
  - 调用 AliDNS OpenAPI（签名请求）进行记录查询/更新/新增。
  - `RECORD_ID` 为空时自动查询子域名记录；找不到则自动新增。

3. 稳定性修复
- 修复了配置校验函数 `require_conf` 在 `set -e` 下的提前退出问题，避免脚本在参数检查阶段误失败。

4. 测试服务器落地
- 测试服务器：`10.0.0.18`（Linux）。
- 已将脚本与配置同步到：`/root/coitd/scripts/cfst_ddns/`。
- 已定位并补齐运行所需二进制：`/opt/cfst-collector/bin/cfst`（复制到运行目录）。
- 已补齐运行依赖输入文件 `ip.txt`（放置于运行目录）。

## 3. 实测过程与结果

1. 初次失败原因（已定位）
- `FOLDER` 路径下无 `cfst` 可执行文件。
- 后续修正后可进入 AliDNS 阶段。

2. 第二次失败原因（已定位）
- 阿里云返回 `InvalidDomainName.NoExist`，原因为配置中的域名写错。

3. 最终成功结果（用户已确认）
- 修正配置后再次执行，CFST 测速与 AliDNS 推送均成功。
- 记录已在阿里云控制台可见并更新成功。
- 本次成功写入记录信息：
  - 主机记录：`cfcnd`
  - 域名：`cokeblog.cn`
  - 类型：`A`
  - 线路：`telecom`
  - 解析值：`162.159.5.40`

## 4. 当前状态结论

1. DDNS 迁移到 AliDNS 已跑通，具备可用性。
2. 测速 -> 取最优 IP -> 推送阿里云解析 的链路已验证可闭环。
3. 后续可以进入网页侧参数化配置与任务可视化联动阶段。

## 5. 下一阶段建议

1. 增加“线路枚举/校验”能力（支持通过 API 查询可用 `Line`，避免手填错误）。
2. 在网页侧增加 DDNS 配置管理页（不展示密钥明文，支持掩码与更新）。
3. 将 DDNS 执行结果接入现有任务日志面板，形成统一可观测入口。
