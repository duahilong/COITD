# DDNS 多 IP 推送能力测试记录（2026-04-19）

## 1. 本次改动

脚本：`scripts/cfst_ddns/cfst_ddns.sh`

新增能力：
- 支持通过配置项 `PUSH_IP_COUNT` 控制推送 IP 数量（默认 `1`）。
- 从 CFST 结果中按顺序读取 Top N IP，并逐条线路（`LINE`）逐个 upsert 到 AliDNS。
- 保留原有多线路能力（如 `LINE=telecom,unicom`）。
- `RECORD_ID` 仅在“单线路 + `PUSH_IP_COUNT=1`”场景生效，其余场景自动忽略并按线路自动匹配记录。

示例：

```conf
LINE=telecom,unicom
PUSH_IP_COUNT=3
```

## 2. 测试环境

- 服务器：`10.0.0.18`
- 目录：`/root/coitd/scripts/cfst_ddns`
- 配置：
  - `LINE=telecom,unicom`
  - `PUSH_IP_COUNT=3`（测试时临时写入）

## 3. 测试轮次与结果

### 3.1 第一轮（新增记录验证）

- 退出码：`0`
- 关键日志：
  - `Resolved PUSH_IP_COUNT: 3`
  - `Resolved LINE values: telecom,unicom`
  - `Selected IP count: 3`
  - `Selected IP list: 172.64.229.232,162.159.38.76,172.64.52.250`
  - `Line=telecom`：1 条更新 + 2 条新增，全部成功
  - `Line=unicom`：1 条更新 + 2 条新增，全部成功

### 3.2 第二轮（连续运行更新验证）

- 退出码：`0`
- 关键日志：
  - `Resolved PUSH_IP_COUNT: 3`
  - `Resolved LINE values: telecom,unicom`
  - `Selected IP count: 3`
  - `Selected IP list: 172.64.229.105,172.64.53.254,172.64.52.211`
  - `Line=telecom`：匹配到 3 条既有记录并全部更新成功
  - `Line=unicom`：匹配到 3 条既有记录并全部更新成功

## 4. 日志文件位置（测试机）

- 第一轮：`/tmp/cfst_ddns_multiip_round1.log`
- 第二轮：`/tmp/cfst_ddns_multiip_round2.log`

## 5. 结论

`PUSH_IP_COUNT` 多 IP 推送能力已生效，可在单次任务中选出并推送 2~3 个（或更多）IP 到 AliDNS，且与多线路 `LINE` 配置兼容。
