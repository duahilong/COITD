# LINE 参数多配置支持说明（2026-04-19）

## 1. 目标

将 `scripts/cfst_ddns/cfst_ddns.sh` 中的 `LINE` 参数从“仅支持单值”升级为“支持多值配置”，用于一次执行中按多条解析线路同步更新阿里云 DNS。

## 2. 已完成改动

脚本文件：`scripts/cfst_ddns/cfst_ddns.sh`

1. `LINE` 解析增强
- 支持逗号分隔多值：`LINE=telecom,unicom,mobile,edu`
- 自动去空格、去重
- 若配置为空，自动回退为 `default`

2. 更新逻辑改为按线路逐条处理
- 对每个 `Line` 独立执行：
  - 先查记录（`DescribeSubDomainRecords`）
  - 找到则更新（`UpdateDomainRecord`）
  - 找不到则新增（`AddDomainRecord`）

3. `RECORD_ID` 兼容规则
- 单线路时：若配置了 `RECORD_ID`，优先使用
- 多线路时：自动忽略单个 `RECORD_ID`，改为按线路自动匹配对应记录，避免写错线路记录

4. 配置模板说明更新
- `scripts/cfst_ddns/cfst_ddns.conf.example` 已补充注释，明确 `LINE` 支持多值格式

## 3. 使用方式

示例一（单线路）：

```conf
LINE=default
```

示例二（多线路）：

```conf
LINE=telecom,unicom,mobile,edu
```

示例三（多线路，含空格，脚本会自动处理）：

```conf
LINE=telecom, unicom , mobile ,edu
```

## 4. 运行行为说明

当配置多线路时，脚本会在一次运行中按线路依次输出：

- `Querying AliDNS record id ... (Line=xxx)`
- `Updating AliDNS record for Line=xxx ...` 或 `creating new AliDNS record`
- `AliDNS upsert success. Line=xxx ...`

这意味着“一个线路对应一条记录”，而不是把多个线路塞进同一条记录。

## 5. 测试结论（测试机 10.0.0.18）

已使用多线路配置进行实测，关键结果如下：

1. `LINE=telecom,unicom,mobile,edu`：4 条线路均成功 upsert  
2. `LINE=telecom,unicom`：2 条线路连续测试成功，退出码为 `0`

结论：`LINE` 多配置已可用，能够稳定按多线路执行更新。

## 6. 注意事项

1. `LINE` 需要填写阿里云 DNS 支持的线路编码（如 `default`、`telecom`、`unicom`、`mobile`、`edu`）。
2. 若填入不支持的线路编码，AliDNS 会返回错误码，脚本会直接报错退出并输出 API 错误信息。
