# COITD 跨机器部署教程（Linux 全流程）

> 适用对象：需要把本项目部署到新的 Linux 服务器（不仅是测试机）
> 
> 最终目标：脚本可手动运行、可定时运行、可网页查看状态与结果

## 1. 部署前准备

### 1.1 系统要求
1. Linux（建议 Ubuntu 22.04+ / Debian 12+ / CentOS 7+）
2. 已安装并可用：`bash`、`python3`、`git`、`systemd`
3. 机器可访问 AliDNS OpenAPI（外网）

### 1.2 端口与权限
1. Web 页面默认端口：`8091`
2. 需要 root 或有 sudo 权限（用于写 systemd 单元和启动服务）

### 1.3 目录规划（建议）
以下教程以 `/root/coitd` 为例，你也可以改成自己的目录。

## 2. 拉取项目代码

```bash
cd /root
git clone <你的仓库地址> coitd
cd /root/coitd
```

如果已经有代码仓库，可改为 `git pull` 更新即可。

### 2.1 可选：旧部署清空（重装场景）

如果机器上已经部署过旧版本，建议先清理：

```bash
sudo systemctl stop cfst-ddns.timer cfst-ddns.service cfst-ddns-web.service 2>/dev/null || true
sudo systemctl disable cfst-ddns.timer cfst-ddns-web.service 2>/dev/null || true
sudo rm -f /etc/systemd/system/cfst-ddns.timer /etc/systemd/system/cfst-ddns.service /etc/systemd/system/cfst-ddns-web.service
sudo systemctl daemon-reload

rm -rf /root/coitd/scripts/cfst_ddns/state /root/coitd/scripts/cfst_ddns/logs
rm -f /root/coitd/scripts/cfst_ddns_web/ddns_web_server.log
```

## 3. 准备 CFST 二进制

脚本依赖 `cfst` 可执行文件。请确认 `cfst` 存在并可执行。

推荐检查命令：

```bash
ls -l /root/coitd/CloudflareSpeedTest/cfst
file /root/coitd/CloudflareSpeedTest/cfst

# 如果上面没有，再检查脚本目录（测试机就是这个路径）
ls -l /root/coitd/scripts/cfst_ddns/cfst
file /root/coitd/scripts/cfst_ddns/cfst
```

`FOLDER` 必须填写为“包含 `cfst` 可执行文件的目录”，不是固定写法。

## 4. 配置 DDNS 脚本

### 4.1 初始化配置文件

```bash
cd /root/coitd/scripts/cfst_ddns
cp -f cfst_ddns.conf.example cfst_ddns.conf
```

### 4.2 编辑配置

```bash
vim /root/coitd/scripts/cfst_ddns/cfst_ddns.conf
```

推荐最小配置示例：

```ini
FOLDER=/root/coitd/scripts/cfst_ddns
ALI_ACCESS_KEY_ID=你的AK
ALI_ACCESS_KEY_SECRET=你的SK
DOMAIN_NAME=example.com
RR=www
TYPE=A
TTL=600
LINE=telecom,unicom
PUSH_IP_COUNT=2
ENABLE_SUMMARY=true
SUMMARY_DIR=state
RECORD_ID=
CFST_ARGS=-n 100 -t 4 -dn 10 -dt 10 -httping -cfcolo NRT,KIX
RESULT_FILE=result_ddns.txt
```

如果你的 `cfst` 在其他目录（例如 `/root/coitd/CloudflareSpeedTest`），把 `FOLDER` 改为对应目录即可。

### 4.3 参数说明（关键）
1. `LINE`：支持多值，逗号分隔（如 `telecom,unicom,mobile,default`）
2. `PUSH_IP_COUNT`：每条线路推送的 IP 数（`1~20`）
3. `RECORD_ID`：仅在“单线路 + PUSH_IP_COUNT=1”时建议使用；多线路/多 IP 会自动忽略
4. `ENABLE_SUMMARY`：建议保持 `true`，Web 页面依赖 summary 文件
5. `SUMMARY_DIR`：默认 `state`，会产生 `latest.json` 与 `history/*.json`

## 5. 首次手动验证（强烈建议）

### 5.1 执行脚本

```bash
cd /root/coitd/scripts/cfst_ddns
bash ./cfst_ddns.sh
```

### 5.2 验证输出

```bash
# 查看运行日志
ls -l /root/coitd/scripts/cfst_ddns/logs

# 查看 summary 最新结果
cat /root/coitd/scripts/cfst_ddns/state/latest.json

# 查看历史文件
ls -lt /root/coitd/scripts/cfst_ddns/state/history | head
```

你应重点确认：
1. `status` 是否为 `success`
2. `selected_ips` 数量是否符合 `PUSH_IP_COUNT`
3. `ddns.operations` 中各线路的 `rank/total` 是否正确（如 `1/2`,`2/2`）

## 6. 配置 systemd（定时运行）

### 6.1 创建 service 单元

创建 `/etc/systemd/system/cfst-ddns.service`：

```ini
[Unit]
Description=COITD DDNS one-shot update
Wants=network-online.target
After=network-online.target

[Service]
Type=oneshot
User=root
Group=root
WorkingDirectory=/root/coitd/scripts/cfst_ddns
ExecStartPre=/usr/bin/mkdir -p /root/coitd/scripts/cfst_ddns/logs
ExecStart=/usr/bin/flock /run/lock/cfst-ddns.lock /bin/bash -c './cfst_ddns.sh >> /root/coitd/scripts/cfst_ddns/logs/cfst_ddns_run.log 2>&1'
TimeoutStartSec=30min
Nice=5
```

> `flock` 保证同一时间只有一个实例运行，避免并发冲突。

### 6.2 创建 timer 单元

创建 `/etc/systemd/system/cfst-ddns.timer`（示例：每 1 小时）：

```ini
[Unit]
Description=Run COITD DDNS updater periodically

[Timer]
OnBootSec=3min
OnUnitActiveSec=1h
Persistent=true
AccuracySec=1min
RandomizedDelaySec=30s
Unit=cfst-ddns.service

[Install]
WantedBy=timers.target
```

### 6.3 生效并启动

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now cfst-ddns.timer
```

### 6.4 验证状态

```bash
systemctl status cfst-ddns.timer --no-pager
systemctl status cfst-ddns.service --no-pager
systemctl list-timers --all | grep cfst-ddns
```

## 7. 配置 Web 监控页

### 7.1 手动启动（先验证）

```bash
cd /root/coitd
python3 scripts/cfst_ddns_web/ddns_web_server.py \
  --host 0.0.0.0 \
  --port 8091 \
  --timer-name cfst-ddns.timer \
  --service-name cfst-ddns.service \
  --state-dir /root/coitd/scripts/cfst_ddns/state \
  --run-log-file /root/coitd/scripts/cfst_ddns/logs/cfst_ddns_run.log
```

浏览器访问：`http://<服务器IP>:8091`

### 7.2 建议改为 systemd 管理 Web

创建 `/etc/systemd/system/cfst-ddns-web.service`：

```ini
[Unit]
Description=COITD DDNS Web Console
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=root
Group=root
WorkingDirectory=/root/coitd
ExecStart=/usr/bin/python3 /root/coitd/scripts/cfst_ddns_web/ddns_web_server.py --host 0.0.0.0 --port 8091 --timer-name cfst-ddns.timer --service-name cfst-ddns.service --state-dir /root/coitd/scripts/cfst_ddns/state --run-log-file /root/coitd/scripts/cfst_ddns/logs/cfst_ddns_run.log
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启动并设开机自启：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now cfst-ddns-web.service
systemctl status cfst-ddns-web.service --no-pager
```

## 8. 常用运维命令

### 8.1 手动跑一轮

```bash
systemctl start cfst-ddns.service
```

### 8.2 查看日志

```bash
# 脚本累计日志
tail -n 120 /root/coitd/scripts/cfst_ddns/logs/cfst_ddns_run.log

# Web 服务日志
journalctl -u cfst-ddns-web.service -n 120 --no-pager

# 定时服务日志
journalctl -u cfst-ddns.service -n 120 --no-pager
```

### 8.3 修改定时频率

编辑 `/etc/systemd/system/cfst-ddns.timer` 的 `OnUnitActiveSec`：
1. 每半小时：`30min`
2. 每1小时：`1h`
3. 每4小时：`4h`
4. 每12小时：`12h`

改完执行：

```bash
sudo systemctl daemon-reload
sudo systemctl restart cfst-ddns.timer
systemctl list-timers --all | grep cfst-ddns
```

## 9. 重要行为说明

1. 页面“立即运行脚本”按钮等价于 `systemctl start cfst-ddns.service`。
2. 当前 timer 采用 `OnUnitActiveSec`，手动运行一次后，下次自动执行时间会相应后移（这是 systemd 设计行为）。
3. 页面“自动更新时间”是 UI 秒级刷新；后端状态仍低频拉取，不会每秒读取日志。

## 10. 常见问题排查

### 10.1 页面显示只有 1 个 IP
排查顺序：
1. 检查配置 `PUSH_IP_COUNT` 是否已设置为 2/3
2. 查看 `state/latest.json` 的 `config.push_ip_count`
3. 看 `selected_ips` 数组长度与 `ddns.operations` 的 `rank/total`

### 10.2 阿里云返回线路错误
1. 检查 `LINE` 拼写（如 `telecom,unicom,mobile,default`）
2. 查看脚本日志中的 API 错误码和错误信息

### 10.3 定时任务不执行
1. `systemctl status cfst-ddns.timer`
2. `systemctl list-timers --all | grep cfst-ddns`
3. `journalctl -u cfst-ddns.service -n 200 --no-pager`

### 10.4 Web 打不开
1. `systemctl status cfst-ddns-web.service`
2. `ss -lntp | grep 8091`
3. 检查安全组/防火墙是否放通 8091

## 11. 升级流程（后续机器复用）

```bash
cd /root/coitd
git pull
sudo systemctl restart cfst-ddns-web.service
# 如果脚本逻辑变更涉及定时器/服务定义，再执行
sudo systemctl daemon-reload
sudo systemctl restart cfst-ddns.timer
```

## 12. 建议的标准验收清单

1. `cfst_ddns.sh` 手动执行成功
2. `state/latest.json` 正常更新
3. `cfst-ddns.timer` 正常激活，能看到下一次运行时间
4. Web 页面可访问，状态与 summary 一致
5. AliDNS 后台能看到对应记录更新
