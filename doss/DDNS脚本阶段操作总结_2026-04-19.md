# DDNS 脚本阶段操作总结（2026-04-19）

## 1. 阶段目标

将 `cfst_ddns` 从 Cloudflare 推送方式迁移到阿里云 DNS（AliDNS），并在测试机完成可重复验证。

## 2. 关键改动

1. 新建独立工作目录
- 目录：`scripts/cfst_ddns/`
- 目的：与 `CloudflareSpeedTest` 只读目录隔离，避免修改上游源码目录。

2. DDNS 逻辑迁移到阿里云
- 脚本：`scripts/cfst_ddns/cfst_ddns.sh`
- 配置：`scripts/cfst_ddns/cfst_ddns.conf`
- 实现能力：
  - 运行 `cfst` 获取最佳 IP；
  - 通过 AliDNS OpenAPI 查询记录（`DescribeSubDomainRecords`）；
  - 有 `RECORD_ID` 时执行更新（`UpdateDomainRecord`）；
  - 无记录时自动新增（`AddDomainRecord`）。

3. 配置参数修正
- `CFST_ARGS` 从错误写法改为可生效写法：
  - `--cfcolo` -> `-cfcolo`
  - 增加 `-httping`，使地区过滤参数生效
- 当前示例：
  - `CFST_ARGS=-n 100 -t 4 -dn 10 -dt 10 -httping -cfcolo NRT`

4. 脚本 bug 修复
- 修复前错误：
  - 报错：`NameError: name 'false' is not defined`
- 根因：
  - 脚本使用了 `python3 - <<'PY' <<<"${resp}"`，导致 JSON 输入与 HereDoc 冲突，Python 将 JSON 当代码执行。
- 修复方式：
  - 改为 `printf '%s' "${resp}" | python3 -c '...'` 解析 JSON，避免 stdin 冲突。

## 3. 测试机验证

测试机信息：
- `10.0.0.18`
- 用户：`root`

验证结论：
1. 脚本可完整执行至 DDNS 推送阶段；
2. AliDNS 记录可成功创建/更新；
3. 修复后可稳定拿到 `RequestId` 与 `RecordId`，无 `NameError` 异常。

## 4. 当前状态

1. 脚本链路“CFST 测速 -> 取最佳 IP -> 推送 AliDNS”已跑通；
2. 现阶段可继续做高频回归测试（多次运行）和网页端联动；
3. 后续可新增“可用线路自动查询/校验（Line）”以降低配置错误概率。
